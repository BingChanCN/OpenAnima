# Update Spec Language Convention

## Goal
Clarify the project's collaboration language convention so AI assistants communicate with users in Chinese while keeping spec documents written in English.

## Requirements
- Add a clear convention to the spec documentation entry points.
- Preserve the existing rule that spec documentation is written in English.
- Make the collaboration language expectation easy to discover during session startup.

## Acceptance Criteria
- [ ] The relevant `.trellis/spec/` index files state that agent-user conversation should be in Chinese.
- [ ] The same files state that spec documentation must remain in English.
- [ ] The wording is consistent across the updated files.

## Technical Notes
This is a documentation-only change. No runtime or behavior changes are expected.
