    - Test: AnimaRoute_AwaitResponse_PublishesToResponsePort — simulate successful RouteResult, assert response published to {ModuleName}.port.response
    - Test: AnimaRoute_OnTimeout_OutputsErrorJson — simulate RouteResult.Failed(Timeout), assert error port receives JSON with {"error":"Timeout","target":"animaB::summarize","timeout":30}
    - Test: AnimaRoute_OnNotFound_OutputsErrorJson — simulate RouteResult.Failed(NotFound), assert error port receives JSON with {"error":"NotFound","target":...}
    - Test: AnimaRoute_ResponseAndError_MutuallyExclusive — on success, error port NOT triggered; on failure, response port NOT triggered
    - Integration Test: AnimaRoute_E2E_FullRoundTrip — create two EventBuses (Anima A and B), wire AnimaInputPort on B, AnimaOutputPort on B (with a FixedText in between to simulate processing), AnimaRoute on A. Fire trigger on A, verify response arrives at A's response port.
  </behavior>
  <action>
    1. Create `src/OpenAnima.Core/Modules/AnimaRouteModule.cs`:
       - Attributes: `[InputPort("request", PortType.Text)]`, `[InputPort("trigger", PortType.Trigger)]`, `[OutputPort("response", PortType.Text)]`, `[OutputPort("error", PortType.Text)]`
       - Implements `IModuleExecutor`
       - Constructor: `IEventBus eventBus, ICrossAnimaRouter router, IAnimaModuleConfigService configService, IAnimaContext animaContext, ILogger<AnimaRouteModule> logger`
       - Metadata: `new ModuleMetadataRecord("AnimaRouteModule", "1.0.0", "Sends a request to a remote Anima and awaits the response")`
       - Private field `_lastRequestPayload` (string?) to buffer the most recent request data.
       - `InitializeAsync`:
         a. Subscribe to `{Metadata.Name}.port.request` — store payload in `_lastRequestPayload`
         b. Subscribe to `{Metadata.Name}.port.trigger` — call `HandleTriggerAsync`
       - `HandleTriggerAsync` (CRITICAL — MUST await):
         a. Read config for `targetAnimaId` and `targetPortName`
         b. If `_lastRequestPayload` is null, publish error JSON `{"error":"NoRequestData","target":"..."}` to error port and return
         c. `var result = await _router.RouteRequestAsync(targetAnimaId, targetPortName, _lastRequestPayload, timeout: TimeSpan.FromSeconds(30), ct)`
         d. If `result.IsSuccess`: publish `result.Payload` to `{Metadata.Name}.port.response`
         e. If NOT success: publish JSON error to `{Metadata.Name}.port.error`:
            ```csharp
            var errorJson = JsonSerializer.Serialize(new {
                error = result.Error.ToString(),
                target = $"{targetAnimaId}::{targetPortName}",
                timeout = 30
            });
            ```
         f. response and error are MUTUALLY EXCLUSIVE — only one publishes per trigger
       - `ExecuteAsync`: No-op (event-driven via trigger subscription).
       - `ShutdownAsync`: Dispose subscriptions, clear `_lastRequestPayload`.

    2. Add unit tests to `tests/OpenAnima.Tests/Modules/RoutingModulesTests.cs` covering all unit test behaviors above. For the router mock in unit tests, create a simple `TestCrossAnimaRouter` that records calls and returns configurable RouteResult values. Use real EventBus for event flow.

    3. Create `tests/OpenAnima.Tests/Integration/CrossAnimaRoutingE2ETests.cs` with `[Trait("Category", "Routing")]`:
       - E2E test creates a real CrossAnimaRouter with IAnimaRuntimeManager mock
       - Creates two separate EventBus instances (Anima A and Anima B)
       - Wires AnimaInputPort on B's EventBus, subscribes to B's output, and has a handler that publishes back to AnimaOutputPort's input (simulating a processing pipeline)
       - Wires AnimaRoute on A's EventBus
       - Fires trigger on A, asserts response arrives at A's response port within 5 seconds
       - This proves the full round-trip: A trigger → CrossAnimaRouter → B EventBus → AnimaInputPort output → (simulated processing) → AnimaOutputPort → CompleteRequest → A response port
  </action>
