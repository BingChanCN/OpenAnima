# Plan 66-03: ChatPanel history integration, LLM context slicing, and Settings UI

## Objective

Wire up the ChatHistoryService into ChatPanel so that:
1. Each user message and assistant message are persisted to chat.db as they occur.
2. On Anima activation (startup or Anima switch), chat history is restored from the DB into `ChatSessionState.Messages`.
3. When sending a message to the LLM, a token-budget-based slice of history is passed via the `messages` port instead of the bare user message via `SendMessageAsync`, so the LLM receives conversation context.
4. Settings page gains a "Chat context budget" numeric field (key: `chatContextBudget`, service: `ChatPersistence`, default: `4000`) configurable per Anima.

## must_haves

- After user sends a message, `IChatHistoryService.SaveMessageAsync(animaId, "user", content, null, 0, 0)` is called
- After assistant stream completes, `IChatHistoryService.FinalizeLastMessageAsync(animaId, finalContent)` is called (the streaming row was inserted by `SaveMessageAsync(..., isStreaming: true)` at stream start)
- On `OnAnimaChanged` / `OnInitializedAsync`, all rows from `GetMessagesAsync(animaId)` are loaded into `ChatSessionState.Messages`
- On startup, `MarkInterruptedAsync(animaId)` is called for the active Anima before history is loaded into ChatPanel
- The LLM receives history via `messages` port: slice uses token-budget walk from newest to oldest, stopping when accumulated `input_tokens + output_tokens` from DB rows exceeds `chatContextBudget`; included messages are sent newest-first then reversed to chronological order for the LLM
- Settings page shows a "Chat context budget" field that reads/writes `chatContextBudget` in the `ChatPersistence` config namespace per Anima
- Default `chatContextBudget` = `4000` (tokens) when not configured

<action>
The current flow: `ChatPanel.SendMessage` calls `_chatInputModule.SendMessageAsync(prompt)`, which fires `ChatInputModule.port.userMessage`. LLMModule receives this on the `prompt` port and builds a single-turn exchange.

For history slicing, we need to send the full conversation slice via the `messages` port of LLMModule. `ChatInputModule` fires on `port.userMessage` — this reaches LLMModule's `prompt` port. To use LLMModule's `messages` port, we need to publish a serialized `List<ChatMessageInput>` to the `messages` port event name.

Modify `src/OpenAnima.Core/Components/Shared/ChatPanel.razor`:

1. Add a private method `BuildContextSlice(string animaId, string userMessage, List<ChatSessionMessage> priorMessages)` that:
   a. Reads `chatContextBudget` from config: `_moduleConfigService.GetConfig(animaId, "ChatPersistence")["chatContextBudget"]`, default `4000` if missing or invalid
   b. Receives `priorMessages` as a parameter (messages captured BEFORE the new user message was added — clean, no filtering needed)
   c. Walks from NEWEST to OLDEST in `priorMessages`, accumulating token count from `_contextManager.CountTokens(m.Content)`, stopping when total exceeds `chatContextBudget`
   d. Reverses the accumulated list to chronological order
   e. Appends the current user message at the end as `new ChatMessageInput("user", userMessage)`
   f. Returns the resulting `List<ChatMessageInput>`

   ```csharp
   private List<ChatMessageInput> BuildContextSlice(string animaId, string userMessage, List<ChatSessionMessage> priorMessages)
   {
       const int DefaultBudget = 4000;
       var config = _moduleConfigService.GetConfig(animaId, "ChatPersistence");
       var budget = config.TryGetValue("chatContextBudget", out var budgetStr) &&
                    int.TryParse(budgetStr, out var parsed) ? parsed : DefaultBudget;

       // Walk newest to oldest, accumulate within budget
       var slice = new List<ChatSessionMessage>();
       int accumulated = 0;
       for (int i = priorMessages.Count - 1; i >= 0; i--)
       {
           var tokens = _contextManager.CountTokens(priorMessages[i].Content);
           if (accumulated + tokens > budget) break;
           accumulated += tokens;
           slice.Insert(0, priorMessages[i]);
       }

       // Convert to ChatMessageInput and append current user message
       var result = slice
           .Select(m => new ChatMessageInput(m.Role, m.Content))
           .ToList();
       result.Add(new ChatMessageInput("user", userMessage));
       return result;
   }
   ```
