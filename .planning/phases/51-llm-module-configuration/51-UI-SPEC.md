## Component Inventory

All components are extensions of existing Blazor scoped-CSS patterns. No new design primitives.

### Provider Dropdown (new field type: CascadingDropdown — first tier)

The Provider dropdown is the **primary visual anchor and entry point** for the config form. It is the first and widest interactive element in the sidebar panel; all downstream fields (Model dropdown or manual fields) are gated behind a selection here.

- Rendered as `<select>` inside `.config-field` wrapper
- Width: 100% of `.config-field` container (sidebar is 380px; field fills content area minus 48px padding = ~332px effective)
- Option format: `{DisplayName} ({N} models)` — e.g., `OpenAI (3 models)`
- Disabled provider option: greyed with `color: var(--text-muted)` and `(已禁用)` suffix; `disabled` attribute applied; shown only if it is the currently saved selection (then not `disabled`, shown at top with inline warning below)
- Final option group: visual separator (`<option disabled>` with `──────────` text) then `手动配置` option
- Focus ring: `border-color: var(--accent-color)`
- On change: clears Model selection, triggers auto-save debounce (500ms, matching existing pattern)

### Model Dropdown (new field type: CascadingDropdown — second tier)

- Visible only when a registered Provider (not "手动配置") is selected
- Placeholder option: `请选择 Model` (value `""`)
- Option format: `{modelId}` when no alias; `{modelId} ({displayAlias})` when alias exists — e.g., `gpt-4o (智能模型)`
- Disabled model option: same treatment as disabled provider — greyed, `(已禁用)` suffix, `disabled` attribute unless it is the currently saved value
- On change: triggers auto-save debounce (500ms)

### Manual Configuration Fields (existing, conditional visibility)

- Visible only when "手动配置" is selected in Provider dropdown
- Three fields: `apiUrl` (text), `apiKey` (password/secret), `modelName` (text)
- Behavior identical to pre-Phase 51; no style changes
- Hidden when Provider dropdown shows a registered provider

### Inline Warning Banner (new, contextual)

- Shown below the Provider dropdown when the currently saved provider is disabled
- Shown below the Model dropdown when the currently saved model is disabled
- Style: matches existing `.error-inline` pattern but using `--warning-color` instead of `--error-color`
  - `background-color: rgba(251, 191, 36, 0.1)`
  - `border-left: 3px solid var(--warning-color)`
  - `color: var(--warning-color)`
  - `font-size: 12px` (0.75rem — consolidated hint/warning size)
  - `padding: 4px 8px` (xs / sm tokens)
  - `border-radius: 0 4px 4px 0`
  - `margin-top: 8px` (sm token)
- Copy: see Copywriting Contract below

### Incomplete Configuration Warning (inline, non-blocking)

- Shown when Provider is selected but Model is still at placeholder (`""`)
- Same `.warning-inline` style as above
- Dismissed automatically when a Model is selected

## Interaction Contracts

### Configuration Mode Switch (Provider → Manual)

| Trigger | Result |
|---------|--------|
| User selects a registered Provider | Show Model dropdown below Provider dropdown; hide manual fields (apiUrl, apiKey, modelName) |
| User selects "手动配置" in Provider dropdown | Hide Model dropdown; show manual fields (apiUrl, apiKey, modelName) |
| User switches Provider from Provider A to Provider B | Clear Model selection to placeholder; Model dropdown repopulates from Provider B's models |
| User switches from registered Provider back to placeholder `""` | Clear Model selection; both dropdowns show at unselected state |

### Auto-Save Debounce

- Delay: 500ms after last change (matches existing `TriggerAutoSave` pattern)
- Triggers: any dropdown `@onchange` event (Provider select, Model select, manual fields `@oninput`)
- On save: show toast `已保存配置` for 2000ms then hide (matches existing `.toast-notification.visible` pattern)
- Skipped: if validation error exists (incomplete config state does NOT block saving in this phase per CONTEXT.md decision)

### Unavailable State Transitions

| State | Display |
|-------|---------|
| Provider deleted (after save) | Auto-clear provider slug from config; Provider dropdown resets to placeholder |
| Provider disabled (after save) | Retain saved slug; show provider name + "(已禁用)" as current value; show inline warning below dropdown; option is rendered but non-selectable for new selection |
| Model deleted (after save) | Auto-clear model ID from config; Model dropdown resets to placeholder |
| Model disabled (after save) | Retain saved model ID; show model label + "(已禁用)" as current value; show inline warning below Model dropdown |

### No-Interaction States

- LLM module with no provider ever configured: Provider dropdown shows at placeholder `请选择 Provider`; manual fields hidden; no warning shown
- LLM module that has only legacy global appsettings config (no per-Anima config): Provider dropdown at placeholder; manual fields hidden; resolution falls through to global at runtime (no UI indicator needed per CONTEXT.md)

|---------|-----------|---------|

| Provider dropdown placeholder | 请选择 Provider | `Editor.LLM.ProviderPlaceholder` |
| Manual option in Provider dropdown | 手动配置 | `Editor.LLM.ManualOption` |
| Divider in Provider dropdown | ────────── | (rendered as disabled `<option>`, no i18n key) |
| Model dropdown label | 模型 | `Editor.LLM.Model` |
| Model dropdown placeholder | 请选择 Model | `Editor.LLM.ModelPlaceholder` |
| Inline warning — disabled provider | 该提供商已禁用，保存的选择仍有效但不会被使用 | `Editor.LLM.ProviderDisabledWarning` |
| Inline warning — disabled model | 该模型已禁用，保存的选择仍有效但不会被使用 | `Editor.LLM.ModelDisabledWarning` |
| Incomplete config warning | 未选择模型，保存后此模块将无法运行 | `Editor.LLM.ModelNotSelectedWarning` |
| Auto-save toast | 已保存配置 | `Editor.Config.SavedToast` (existing key, reuse) |
| Manual config section hint | 手动配置将覆盖全局设置，但优先级低于已选择的提供商 | `Editor.LLM.ManualHint` |
| apiUrl field label | API 地址 | `Editor.LLM.ApiUrl` |
| apiKey field label | API 密钥 | `Editor.LLM.ApiKey` |
| modelName field label | 模型名称 | `Editor.LLM.ModelName` |

Empty state (no providers registered):
- Provider dropdown option (only entry): `尚无已注册的提供商，请前往设置添加`
- i18n key: `Editor.LLM.NoProvidersRegistered`
- The option is `disabled` and is the placeholder value; no other options shown (except "手动配置")

Error state (runtime — no copywriting needed in sidebar):
- Runtime resolution failures surface in the module status section (existing `status-error` + error message pattern), not in the config form
- No new error copy required in this phase

Destructive actions in this phase: None. Module deletion is an existing EditorConfigSidebar action (trash icon in header), unchanged by this phase.
