Integrate CrossAnimaRouter into the application lifecycle: DI registration, AnimaRuntimeManager deletion hooks, EventBus isolation verification, and integration tests.

Purpose: Without lifecycle integration, the router exists in isolation. This plan wires it into the deletion path (so pending requests fail immediately on Anima deletion, ports are cleaned up) and registers it in DI (so modules can access it). The ANIMA-08 isolation test proves cross-Anima routing MUST go through CrossAnimaRouter, not EventBus.

Output: CrossAnimaRouter fully integrated into DI and Anima lifecycle, with integration tests proving deletion behavior and EventBus isolation.
