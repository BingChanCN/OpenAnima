  <action>
    **CrossAnimaRouter.cs:** Implement `ICrossAnimaRouter` in `OpenAnima.Core.Routing` namespace. Constructor takes `ILogger<CrossAnimaRouter>`. Key implementation details:

    Internal state:
    - `_registry`: `ConcurrentDictionary<string, PortRegistration>` with `animaId::portName` compound keys
    - `_pending`: `ConcurrentDictionary<string, PendingRequest>` with correlationId keys
    - `_cleanupCts`: CancellationTokenSource for the cleanup loop
    - `_cleanupTask`: Task running the PeriodicTimer loop

    **RegisterPort:** Build key as `$"{animaId}::{portName}"`. Use `TryAdd` ā€” if false, return DuplicateError. Log at Information level.

    **UnregisterPort:** Build key, call `TryRemove`. Idempotent ā€” no error if not found. Log at Information level.

    **GetPortsForAnima:** Filter `_registry.Values` by `r.AnimaId == animaId`, return `.ToList()`.

    **RouteRequestAsync:**
    1. Build key `$"{targetAnimaId}::{portName}"`. Check `_registry.TryGetValue` ā€” if not found, return `RouteResult.NotFound(key)`.
    2. Generate correlationId: `Guid.NewGuid().ToString("N")` (full 32-char, never truncated).
    3. Create `TaskCompletionSource<RouteResult>(TaskCreationOptions.RunContinuationsAsynchronously)`.
    4. Create linked CTS: `CancellationTokenSource.CreateLinkedTokenSource(ct)`, then `cts.CancelAfter(effectiveTimeout)` where effectiveTimeout = timeout ?? TimeSpan.FromSeconds(30).
    5. Build PendingRequest record, store in `_pending[correlationId]`.
    6. Register cancellation callback on `cts.Token`: on cancel, `TryRemove` from pending, then `TrySetResult` with Timeout (or Cancelled if the external ct was the one that fired).
    7. Log at Debug level: "RouteRequest {CorrelationId} -> {Key}".
    8. **Do NOT deliver to EventBus in this plan** ā€” the router does not take `IAnimaRuntimeManager` yet (that's Plan 02). Instead, simply await `tcs.Task`. The request will either be completed by `CompleteRequest` or time out. Phase 29's AnimaInputPort will handle actual delivery.
    9. Wrap in try/catch for OperationCanceledException ā€” TryRemove and return Timeout.

    **CompleteRequest:** Look up `_pending.TryRemove(correlationId, ...)`. If found, call `pending.Tcs.TrySetResult(RouteResult.Ok(responsePayload, correlationId))`, dispose CTS with try/catch ObjectDisposedException, return true. If not found, return false.

    **CancelPendingForAnima:** Iterate `_pending.Values.Where(p => p.TargetAnimaId == animaId).ToList()`. For each, `TryRemove`, `TrySetResult(RouteResult.Failed(RouteErrorKind.Cancelled, ...))`, dispose CTS with try/catch. Log at Debug level.

    **UnregisterAllForAnima:** Iterate `_registry.Where(kvp => kvp.Value.AnimaId == animaId).ToList()`. For each, `TryRemove`. Log at Information level.

    **Cleanup loop:** Start in constructor via `StartCleanupLoop()`. Use `PeriodicTimer(TimeSpan.FromSeconds(30))` in a `Task.Run` loop (same pattern as HeartbeatLoop). Every tick, find `_pending.Values.Where(p => p.ExpiresAt <= DateTimeOffset.UtcNow)`, `TryRemove` each, `TrySetResult(Timeout)`, dispose CTS. Log at Debug level when entries removed.

    **Dispose:** Cancel `_cleanupCts`, wait for `_cleanupTask` (with try/catch), dispose CTS. Idempotent.

    IMPORTANT anti-patterns to avoid:
    - Never use `SetResult` ā€” always `TrySetResult` (two code paths may race to complete the same TCS)
    - Wrap `cts.Dispose()` in try/catch ObjectDisposedException (CancelPendingForAnima and timeout callback may race)
    - Never use `tcs.Task.Wait()` or `.Result` ā€” always await
    - The cleanup loop is a safety net; primary removal happens in CompleteRequest/CancelPendingForAnima/timeout callback

For timeout tests, use short timeouts (100-200ms) to keep tests fast. For cleanup tests, use a constructor overload or internal method to trigger cleanup manually rather than waiting 30s.
