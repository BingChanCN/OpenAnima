public void ClearMessages_RemovesAllMessages()

## Implementation Notes

**Why this is the minimal correct fix:**
- ChatSessionState is already Scoped (per-circuit), providing baseline isolation across browser tabs
- The gap is within a single circuit: switching Anima does not create a new scope, so messages leak
- Subscribing to ActiveAnimaChanged and calling Messages.Clear() is the minimal fix per 26-RESEARCH.md
- No new service needed — just event subscription in existing ChatPanel component

**Explicitly out of scope:**
- Chat message persistence (STATE.md: "not conversation history")
- Per-Anima message caching (deferred — clear is sufficient for v1.5)
- Chat history storage to disk (not in requirements)

**Pattern consistency:**
- Same event subscription pattern used in AnimaListPanel, ModuleDetailSidebar, Editor.razor
- Same InvokeAsync(StateHasChanged) pattern for cross-thread UI updates
</notes>
