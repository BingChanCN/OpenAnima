## Implementation Notes

**Pattern source:** Direct clone of AnimaModuleStateService (Phase 25-01)
- Same SemaphoreSlim locking pattern
- Same System.Text.Json serialization
- Same per-Anima directory structure

**Key differences from AnimaModuleStateService:**
- Stores Dictionary<string,string> instead of HashSet<string>
- One JSON file per module (not one file for all modules)
- Path: module-configs/{moduleId}.json (not enabled-modules.json)

**Locked decisions from CONTEXT.md:**
- Auto-save behavior (UI concern, not service concern)
- Per-Anima persistence (implemented here)
- Validation (UI concern, service stores any valid Dictionary)

**Deferred to Plan 02:**
- InitializeAsync startup hook (AnimaHostedService)
- UI integration (EditorConfigSidebar)
- Auto-save debounce (component-level concern)
</notes>
