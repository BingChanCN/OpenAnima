Implement the stateless dispatch fork so [StatelessModule]-marked modules execute concurrently, bypassing channel serialization (CONC-07/CONC-08). Stateless modules bypass channel serialization for concurrent execution per the Mixed Dispatch Strategy.

**Modify src/OpenAnima.Core/Anima/AnimaRuntime.cs:**
- Add `using OpenAnima.Core.Channels;`
- Add new public property: `public ActivityChannelHost ActivityChannelHost { get; }`
- In constructor, create ActivityChannelHost with callback lambdas:
  - `onTick`: This is the **stateless dispatch fork** (CONC-07/CONC-08). The callback must NOT simply call `WiringEngine.ExecuteAsync(item.Ct)` — that would serialize all module execution through the channel consumer. Instead:
    1. Get the loaded WiringConfiguration from `WiringEngine.GetCurrentConfiguration()`. If null/not loaded, return.
    2. Partition modules into stateless and stateful groups using `ActivityChannelHost.IsStateless(module)` for each module in the configuration's node list. Look up IModule instances from PluginRegistry by module name.
    3. **Stateless modules:** Fire them concurrently via `Task.WhenAll` — for each stateless module, publish its `"{moduleName}.execute"` event to EventBus directly. These run outside channel serialization.
    4. **Stateful modules:** Call `WiringEngine.ExecuteAsync(item.Ct)` which runs them in topological level order through the channel consumer's serialized path. Note: WiringEngine.ExecuteAsync currently dispatches ALL modules. To avoid double-dispatching stateless modules, either (a) add a filter/skip set to WiringEngine.ExecuteAsync, or (b) accept that stateless modules execute twice (once concurrently, once in WiringEngine) — this is safe since they are stateless by definition, but wasteful. The cleaner approach: pass a `HashSet<string> skipModuleIds` to a new overload `WiringEngine.ExecuteAsync(ct, skipModuleIds)` that skips modules in the set during level iteration. If adding the overload is too invasive, an alternative: tag stateless modules in a thread-local or field before calling ExecuteAsync, and have ExecuteAsync check `ActivityChannelHost.IsStateless` to skip them. **Simplest correct approach:** Add an optional `ISet<string>? skipModuleIds = null` parameter to `WiringEngine.ExecuteAsync`. When non-null, `ExecuteModuleAsync` skips any module whose ID is in the set. This is a 3-line change to WiringEngine.
    5. The stateless modules run concurrently via `Task.WhenAll`, the stateful modules run serialized via `WiringEngine.ExecuteAsync(ct, skipStatelessIds)`, and these two groups can run in parallel: `await Task.WhenAll(statelessTask, statefulTask)`.
  - `onChat`: `async (item) => { await EventBus.PublishAsync(new ModuleEvent<string> { EventName = "ChatInputModule.port.userMessage", SourceModuleId = "ChatInputModule", Payload = item.Message }, item.Ct); }` — publishes to EventBus, letting wiring route the message
  - `onRoute`: `async (item) => { await EventBus.PublishAsync(new ModuleEvent<string> { EventName = item.EventName, SourceModuleId = item.SourceModuleId, Payload = item.Payload, Metadata = item.Metadata }, item.Ct); }` — delivers routing event to this Anima's EventBus
- Call `ActivityChannelHost.Start()` after all runtime components are constructed
- Update DisposeAsync order: `await HeartbeatLoop.StopAsync()` -> `HeartbeatLoop.Dispose()` -> `await ActivityChannelHost.DisposeAsync()` -> `WiringEngine.UnloadConfiguration()`
