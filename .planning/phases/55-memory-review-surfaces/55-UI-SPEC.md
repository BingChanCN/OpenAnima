## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, keyword tag padding, inline badge padding |
| sm | 8px | Row padding, gap between adjacent controls, timeline entry internal padding |
| md | 16px | Section padding, panel padding, collapsible header padding |

## Typography

| Role | Size | Weight | Line Height | Font Stack |
|------|------|--------|-------------|------------|
| Body | 14px | 400 (regular) | 1.5 | sans-serif |
| Label / meta | 12px | 400 (regular) | 1.4 | sans-serif |
| Section heading (card) | 16px | 600 (semibold) | 1.2 | sans-serif |
| Mono (URI, step content) | 12px | 400 (regular) | 1.5 | monospace |

Notes:
- Only two weights are used: 400 and 600 — consistent with existing codebase.
- 14px is the global base (`html, body { font-size: 14px }`); do not use font sizes below 12px.
- Timestamp values use 12px mono (matching `.step-duration` and `.step-time` patterns).
- Diff highlight text inherits body size (14px) — no special size for highlighted lines.
- Sub-group labels ("Outgoing", "Incoming"): 12px weight 600 (consolidated from 11px — below minimum declared size).
- Label badge text: 12px (consolidated from 11px — below minimum declared size).
- Chevron symbol (▶/▼): 12px (consolidated from 10px — below minimum declared size).

Source: `app.css` `html, body` declaration; cross-checked against `StepTimelineRow.razor.css` and `MemoryNodeCard.razor.css`.

## Component Inventory

New elements introduced in this phase, integrated into existing `MemoryNodeCard.razor`:

### 1. Collapsible Section Header
```
[chevron] Section Title        (header row, 44px min-height, cursor: pointer)
```
- Chevron: `▶` (collapsed) / `▼` (expanded) at 12px, color `--text-muted`
- Header text: 12px, weight 600, color `--text-secondary`
- On expand: chevron rotates 180deg via CSS transition `transform 0.15s ease` (matches StepTimelineRow pattern)
- Background: none (inherits card background), no border, bottom border `1px solid --border-color` only when expanded
- Padding: 8px 0 (vertical), no horizontal padding beyond card padding

### 2. Snapshot Timeline Entry (inside "Snapshot History" section)
```
[timestamp]  [content preview ~60 chars…]  [Expand ▶]
```
- Row: padding 8px, border-bottom `1px solid --border-color`, font-size 14px
- Timestamp: 12px mono, color `--text-secondary`, fixed-width column ~160px
- Content preview: 14px, color `--text-muted`, truncated with `text-overflow: ellipsis`
- Expand toggle: text button "Show diff" / "Hide diff", 12px, color `--accent-color`, no border
- Expanded diff block: `background: var(--surface-dark)`, padding 8px, border-radius 4px
  - Added lines: `rgba(74, 222, 128, 0.15)` background + 2px left border `--success-color`
  - Removed lines: `rgba(248, 113, 113, 0.15)` background + 2px left border `--error-color`
  - Unchanged lines: no background, color `--text-secondary`
  - Diff granularity: line-level (not character-level — deferred per CONTEXT.md)
- Restore button: below diff block, "Restore to this version", style matches `btn-cancel` (ghost, `1px solid --border-color`, 14px)
- Newest entry has no "Restore" button (it is the current content)

### 3. Provenance Inline Expansion (inside "Provenance" section)
```
Source: Step [step-id]  [Artifact artifact-id]        [▶ Show details]
```
- When SourceStepId is present: step ID shown as `--accent-color` text, clickable to expand
- Expanded step block: matches StepTimelineRow `.step-detail` pattern — `padding: 8px 16px 16px 24px`, `border-left: 3px solid var(--border-color)`
  - Fields shown: StepType (label+value), StartedAt, CompletedAt, Status (with status color), Output (truncated to ~200 chars, mono 12px in `--surface-dark` block)
- "Manually created" state: pencil symbol `✎` at 12px `--text-muted` + text "Manually created", color `--text-secondary`
- SourceArtifactId (when present): rendered inline as `Artifact [id]` in `--text-secondary`, same row as step info

### 4. Edge Relationship Row (inside "Relationships" section)
```
Outgoing  ─────────────────────────────────────
  This node  →  [counterpart-uri]  [label]  [date]
Incoming   ─────────────────────────────────────
  [counterpart-uri]  →  This node  [label]  [date]
```
- Sub-group labels ("Outgoing", "Incoming"): 12px, weight 600, uppercase, color `--text-muted`, margin-bottom 4px
- Edge row: padding 4px 0, font-size 12px, min-height 32px, border-bottom `1px solid var(--border-color)`
- Arrow separator `→`: color `--text-muted`
- Label badge: inline-block, `background: var(--surface-dark)`, border `1px solid var(--border-color)`, border-radius 3px, padding 2px 6px, font-size 12px, color `--text-secondary`
- Date: 12px mono, `--text-muted`
- Counterpart URI (clickable): 12px mono, color `--accent-color`, cursor pointer, no underline unless hovered
- "This node" text: 12px, color `--text-secondary`, not clickable, no styling

### 5. Edge Hover Tooltip
- Trigger: hover on counterpart URI text
- Popup: `position: absolute`, `background: var(--surface-card)`, `border: 1px solid var(--border-color)`, `border-radius: 4px`, `padding: 8px 12px`, `max-width: 280px`, `z-index: 100`
- Content: first ~100 chars of target/source node content, font-size 12px, color `--text-secondary`, no truncation ellipsis (CSS `white-space: normal`, `word-break: break-word`)
- Animation: `opacity 0 → 1`, `transition: opacity 0.1s ease`, delay 300ms on show
- Positioning: below and right-aligned to the URI text; if near right edge, flip left

### 6. Restore Confirmation Overlay
- Reuse existing `confirm-overlay` + `confirm-dialog` pattern from `MemoryGraph.razor`
- Title: "Restore to this version?"
- Body: "Current content will be saved as a snapshot. This replaces the current node content."
- Buttons: "Restore" (ghost danger style: border `1px solid --error-color`, color `--error-color`) + "Keep current" (`btn-cancel`)
- Clicking outside overlay (backdrop click): cancels, same as existing delete confirm pattern

## Interaction States

| Interaction | Behavior |
|-------------|----------|
| Click collapsible header | Toggle open/closed; chevron animates; section content fades in with `opacity 0→1, 0.15s ease` |
| Click "Show diff" on snapshot | Expand inline diff block below that row; "Show diff" becomes "Hide diff" |
| Click "Restore to this version" | Show restore confirm overlay |
| Confirm restore | Call `IMemoryGraph.WriteNodeAsync` with old snapshot content; show "Restored" flash; reload snapshots |
| Cancel restore | Dismiss overlay, no change |
| Click counterpart URI in edge row | Navigate: select that URI in the left tree and load its detail card (same as `SelectNode(uri)` call pattern) |
| Hover counterpart URI | Show tooltip after 300ms delay; hide on mouse-leave immediately |
| Click "Show details" on provenance step | Fetch StepRecord by ID; render inline step detail block |
| Snapshot section open on node select | Collapsed by default — data fetched only when user expands |
| Relationships section open on node select | Collapsed by default — data fetched only when user expands |
| Provenance section open on node select | Expanded by default — data available from node record, no additional fetch needed |

## Accessibility

- All collapsible section headers: `role="button"`, `aria-expanded="true|false"`, `aria-controls="{section-id}"`
- Collapsible section content: `id="{section-id}"`, `aria-hidden="true|false"` when collapsed
- Snapshot timeline rows: `role="listitem"` within `role="list"`
- Edge rows: `role="listitem"` within `role="list"`
- Counterpart URI links: `role="link"` or `<button>` with `aria-label="Navigate to {uri}"`
- Tooltip: `role="tooltip"`, linked via `aria-describedby` on the trigger
- Restore button: `aria-label="Restore snapshot from {timestamp}"`
- Confirm overlay backdrop: `aria-modal="true"` on dialog, focus trapped inside while open
- Keyboard: collapsible headers respond to Enter/Space; restore confirm responds to Enter (confirm) and Escape (cancel)
