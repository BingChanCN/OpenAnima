    **Constructor dependencies:**
    - `IMemoryGraph _memoryGraph` -- for WriteNodeAsync, QueryByPrefixAsync
    - `IStepRecorder? _stepRecorder` -- optional, for run timeline entries
    - `IAnimaModuleConfigService _configService` -- for sedimentation LLM config (`sedimentProviderSlug`, `sedimentModelId`)

    **For testability:** The class must accept a `Func<IReadOnlyList<OpenAI.Chat.ChatMessage>, CancellationToken, Task<string>>?` constructor parameter named `llmCallOverride`. In production this is null and the service builds its own `ChatClient` from provider config. In tests, this override is used to inject fake LLM responses. This keeps the service unit-testable without mocking the OpenAI SDK.

    **SedimentAsync method flow:**
    1. Record StepRecord start: `_stepRecorder?.RecordStepStartAsync(animaId, "Sedimentation", $"Extracting from exchange (source: {sourceStepId})", null, ct)`
    2. Query existing sediment nodes: `_memoryGraph.QueryByPrefixAsync(animaId, "sediment://", ct)` -- cap at 50 nodes
    3. Build context summary: for each existing node, format as `"- {node.Uri}: {node.Content[..Math.Min(200, node.Content.Length)]}"`, join with newlines
    4. Build extraction prompt:
       - System message: instruct LLM to extract stable knowledge only (facts, preferences, entities, task learnings), not summarize or dump conversation. Include JSON schema for output. Include existing nodes context summary for merge decisions.
       - User message: the conversation formatted as `<conversation>` XML block with role: content pairs
    5. Call extraction LLM (via llmCallOverride if set, otherwise build ChatClient from config)
    6. Parse JSON response into `SedimentationResult` record using `System.Text.Json` with `JsonNamingPolicy.SnakeCaseLower`
    7. If `result.Extracted.Count == 0`: log debug with skipped_reason, record StepComplete with "Skipped: {reason}", return
    8. For each extracted item:
       a. Normalize keywords to JSON array: if value starts with `[`, keep as-is; otherwise split on comma, trim, serialize to JSON array
       b. Generate ID if action is "create" and URI doesn't have one: use `Guid.NewGuid().ToString("N")[..8]` (8-char hex)
       c. Construct MemoryNode with `Uri = item.Uri`, `AnimaId = animaId`, `Content = item.Content`, `DisclosureTrigger = item.DisclosureTrigger`, `Keywords = normalizedKeywords`, `SourceStepId = sourceStepId`, `CreatedAt/UpdatedAt = DateTimeOffset.UtcNow.ToString("O")`
       d. Call `_memoryGraph.WriteNodeAsync(node, ct)` -- this auto-snapshots if URI exists (LIVM-03)
    9. Record StepComplete with `"{count} nodes sedimented: {uri1}, {uri2}, ..."`
    10. Entire body wrapped in try/catch: on exception, log warning, call `_stepRecorder?.RecordStepFailedAsync(stepId, "Sedimentation", ex, ct)`, return without throwing

    ```
    You are a memory extraction assistant. Given a conversation between a user and an AI assistant, extract stable, reusable knowledge: facts about the project, user preferences, named entities, or task learnings. Do NOT summarize the conversation or store ephemeral exchanges like greetings or acknowledgments.

    Return a JSON object with this exact schema:
    {
      "extracted": [
        {
          "action": "create" or "update",
          "uri": "sediment://fact/{id}" or "sediment://preference/{id}" or "sediment://entity/{id}" or "sediment://learning/{id}",
          "content": "single atomic knowledge statement",
          "keywords": "comma,separated,keywords",
          "disclosure_trigger": "single phrase that would trigger recall"
        }
      ],
      "skipped_reason": null or "explanation if nothing was extracted"
    }

    Rules:
    - Each item must be ONE atomic knowledge point
    - Use "update" action and the existing URI when the knowledge refines an existing memory
    - Use "create" action with a descriptive ID when the knowledge is new
    - Set skipped_reason when the conversation contains no stable knowledge
    - Do NOT store raw conversation text or summaries
    ```
