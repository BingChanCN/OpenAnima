### Cross-Phase Integration Issues

1. **Sedimentation LLM Configuration Gap** (LIVM-01, LIVM-02, LIVM-03, LIVM-04)
   - `SedimentationService.CallProductionLlmAsync` reads `sedimentProviderSlug`/`sedimentModelId` from config
   - No UI component, module schema, or config sidebar exposes these keys
   - Feature is silently inert on any fresh deployment

2. **Boot-Memory XML Dead Code** (MEMR-01 — minimal impact)
   - `LLMModule.BuildMemorySystemMessage` has `<boot-memory>` section filtering by `RecallType == "Boot"`
   - `MemoryRecallService.RecallAsync` never produces Boot nodes — only Disclosure and Glossary
   - Boot nodes reach run timeline as StepRecords (requirement met) but not LLM prompt context
   - `RecallPriority("Boot")` priority-0 slot is unreachable

3. **Provider Impact Count Hardcoded** (PROV-03, PROV-04 — UX only)
   - `Settings.razor` lines 184, 205 pass `0` as affected module count in disable/delete confirms
   - Phase 50 SUMMARY noted "Phase 51 will wire actual counts" but Phase 51 did not implement this
   - `ProviderImpactList.razor` component exists and accepts `AffectedModuleCount` — just never receives real data
