## Scope Note

Phase 53 is primarily a backend/service phase. It adds:

1. Tool descriptor injection into the LLM system message (`<available-tools>` XML block in `messages[0]`).
2. Two new `IWorkspaceTool` implementations: `MemoryRecallTool` and `MemoryLinkTool`.
3. Both tools record `StepRecord` entries via the existing `WorkspaceToolModule` dispatcher.

There is **no new page, modal, or dedicated UI surface** in this phase. All visible output flows through existing components:

- **Run timeline (`StepTimelineRow`)** — renders tool call steps already; `memory_recall` and `memory_link` invocations appear as step rows automatically.
- **Memory graph page (`MemoryGraph.razor`)** — edges created by `memory_link` become visible through the existing graph view without new UI.
- **No new Blazor pages or sidebar panels are introduced in this phase.**

The contract below specifies how the two new tool steps must appear within the existing `StepTimelineRow` component, and how tool descriptor presence should be expressed in any future run-detail system-message viewer.

## Step Rendering Contract (New Tool Steps)

The two new tools (`memory_recall`, `memory_link`) emit `StepRecord` entries via the existing `WorkspaceToolModule` dispatcher. They must render in `StepTimelineRow` following the established pattern:

### memory_recall step row

| Field | Value |
|-------|-------|
| Step type label | `memory_recall` |
| Status icon | success-color check on success; error-color on failure |
| Expanded detail — query | Label: `Query` / Value: the query string, body 14px |
| Expanded detail — results | Label: `Matched nodes` / Value: count integer, body 14px |
| Expanded detail — matched URIs | Monospace 12px, one URI per line, `--font-mono`, `--surface-dark` background block |
| Error detail | `step-error` block with `--error-color` left border and monospace error text |

### memory_link step row

| Field | Value |
|-------|-------|
| Step type label | `memory_link` |
| Status icon | success-color check on success; error-color on failure |
| Expanded detail — from | Label: `From` / Value: monospace URI |
| Expanded detail — to | Label: `To` / Value: monospace URI |
| Expanded detail — relationship | Label: `Relationship` / Value: body 14px free-text string |
| Error detail | `step-error` block when source or target node not found |

No new CSS classes are needed — compose from existing `.step-field`, `.step-content`, `.step-error`, and `.step-module` classes already defined in `StepTimelineRow.razor.css`.

## Interaction Contract

### Tool descriptor injection (invisible to user)

- When `WorkspaceToolModule` is enabled for the current Anima: inject `<available-tools>` block into `messages[0]` alongside `<system-memory>` from Phase 52.
- When `WorkspaceToolModule` is not enabled: omit the block entirely. No visible change in any UI.
- No user-facing toggle or indicator is added in this phase (deferred per CONTEXT.md).

### memory_recall execution flow

1. Agent issues `memory_recall` tool call with `query` and `anima_id` parameters.
2. `WorkspaceToolModule` dispatches to `MemoryRecallTool.ExecuteAsync`.
3. `MemoryRecallTool` calls `IMemoryGraph.FindGlossaryMatches` and `DisclosureMatcher.Match`.
4. Returns `ToolResult.Ok` with matched node URIs and content.
5. `StepRecord` emitted — appears in run timeline as a collapsible `step-row`.
6. If no matches: returns `ToolResult.Ok` with empty list (not an error state).

### memory_link execution flow

1. Agent issues `memory_link` tool call with `from_uri`, `to_uri`, `relationship`, and `anima_id` parameters.
2. `WorkspaceToolModule` dispatches to `MemoryLinkTool.ExecuteAsync`.
3. `MemoryLinkTool` validates both nodes exist — if either is missing, returns `ToolResult.Failed` with specific error copy.
4. On validation pass: calls `IMemoryGraph.AddEdgeAsync` with `SourceStepId` and `SourceArtifactId` from current run context.
5. Returns `ToolResult.Ok` confirming edge creation.
6. `StepRecord` emitted — appears in run timeline as a collapsible `step-row`.
7. Edge becomes visible on `/memory` graph view immediately (existing graph rendering; no new UI code).

### No new destructive confirmations

This phase introduces no destructive user-facing actions. All tool operations are agent-executed. The `memory_link` tool creates edges only; no delete or overwrite behavior.
