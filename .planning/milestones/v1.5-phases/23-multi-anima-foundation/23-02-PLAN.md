    1. **AnimaInitializationService.cs** (in Hosting namespace, mirrors OpenAnimaHostedService pattern):
       - Implements IHostedService
       - Constructor injects IAnimaRuntimeManager and IAnimaContext
       - StartAsync: calls AnimaRuntimeManager.InitializeAsync(). If GetAll() is empty after init, calls CreateAsync("Default"). Sets AnimaContext.SetActive to first Anima's ID.
       - StopAsync: no-op (AnimaRuntimeManager handles its own disposal)

    2. **AnimaCreateDialog.razor** (extends ConfirmDialog pattern with input field):
       - Parameters: IsVisible (bool), OnCreate (EventCallback<string>), OnCancel (EventCallback)
       - Modal with backdrop, title "Create New Anima", text input for name, Cancel + Create buttons
       - Create button disabled when name is empty or whitespace
       - On confirm: invoke OnCreate with trimmed name, clear input
       - CSS: reuse ConfirmDialog.razor.css patterns (modal-backdrop, modal-dialog, modal-header, modal-body, modal-actions). Add input styling.

    3. **AnimaContextMenu.razor** (absolute-positioned context menu):
       - Parameters: IsVisible (bool), X (double), Y (double), OnRename (EventCallback), OnClone (EventCallback), OnDelete (EventCallback), OnClose (EventCallback)
       - Renders when IsVisible: absolute-positioned div at (X, Y) with three buttons: Rename, Clone, Delete (delete has danger class)
       - Backdrop div that closes menu on click
       - CSS: context-menu with background, border-radius, shadow, z-index above sidebar. Buttons styled as menu items.

    4. **AnimaListPanel.razor** (main sidebar component):
       - @implements IAsyncDisposable
       - @inject IAnimaRuntimeManager AnimaManager
       - @inject IAnimaContext AnimaContext
       - Subscribe to AnimaManager.StateChanged and AnimaContext.ActiveAnimaChanged in OnInitialized. Call InvokeAsync(StateHasChanged) in handlers.
       - Unsubscribe in DisposeAsync (ARCH-06).
       - Receives [CascadingParameter] bool SidebarCollapsed from MainLayout.
       - **Expanded state:** Vertical card list. Each card shows Anima name + status dot (green=active, gray=inactive). Active Anima has highlighted background (var(--active-bg)). Click card → AnimaContext.SetActive(id).
       - **Collapsed state:** Circular avatar showing first character of name. Active has accent border. Click → SetActive.
       - **"+" button** at bottom of list: opens AnimaCreateDialog. On create → AnimaManager.CreateAsync(name), then SetActive to new Anima.
       - **Right-click** on card: show AnimaContextMenu at mouse position (@oncontextmenu with preventDefault).
       - **Rename handler:** prompt-style inline edit (replace card text with input, Enter to confirm, Escape to cancel). Calls AnimaManager.RenameAsync.
       - **Clone handler:** AnimaManager.CloneAsync(id), then SetActive to cloned Anima.
       - **Delete handler:** show ConfirmDialog with "Delete {name}?" message. On confirm → AnimaManager.DeleteAsync. If deleted Anima was active, switch to first remaining Anima or null.
       - CSS: anima-list-section with overflow-y auto, max-height calc. anima-card with padding, border-radius 8px, cursor pointer, transition. anima-card.active with var(--active-bg) background and var(--accent-color) left border. anima-avatar (collapsed) as 36px circle with centered text. add-anima-btn styled as dashed border button.
