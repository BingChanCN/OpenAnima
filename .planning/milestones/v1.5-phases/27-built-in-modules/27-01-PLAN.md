  <action>
Create four new module classes in `src/OpenAnima.Core/Modules/`, each following the exact IModuleExecutor pattern established by LLMModule.cs and ChatOutputModule.cs. All modules inject `IEventBus`, `IAnimaModuleConfigService`, `IAnimaContext`, and `ILogger<T>`. All modules read config at execution time (NOT in InitializeAsync) via `_animaContext.ActiveAnimaId` to ensure correct per-Anima behavior since modules are singletons.

**1. FixedTextModule.cs**
- Ports: `[OutputPort("output", PortType.Text)]` — no static input ports (inputs are dynamic via config variables)
- Metadata: `new ModuleMetadataRecord("FixedTextModule", "1.0.0", "Outputs configurable text with template interpolation")`
- InitializeAsync: Subscribe to `{Metadata.Name}.execute` event from WiringEngine. When the execute event fires, call ExecuteAsync.
- ExecuteAsync:
  1. Read config via `_configService.GetConfig(animaId, Metadata.Name)`
  2. Get `template` key from config (default empty string if missing — do NOT treat empty template as an error)
  3. Get all other config keys as static variables
  4. Interpolate template: replace `{{key}}` with value using `string.Replace($"{{{{{key}}}}}", value)` for each variable
  5. Publish interpolated result to `{Metadata.Name}.port.output` via EventBus
- The module does NOT need input port subscriptions for this plan. Dynamic input ports (receiving port data as variables) are a future enhancement. For now, only static config variables and the execute event trigger are needed.
- Allow empty template output (valid state per CONTEXT.md pitfall #4)

**2. TextJoinModule.cs**
- Ports: `[InputPort("input1", PortType.Text)]`, `[InputPort("input2", PortType.Text)]`, `[InputPort("input3", PortType.Text)]`, `[OutputPort("output", PortType.Text)]` — fixed 3 input ports (satisfies both BUILTIN-03 concat and BUILTIN-05 merge; per RESEARCH.md, dynamic ports are not supported by the static port system)
- Metadata: `new ModuleMetadataRecord("TextJoinModule", "1.0.0", "Joins multiple text inputs into one output")`
- Internal state: `Dictionary<string, string> _receivedInputs = new()` to buffer received port data
- InitializeAsync: Subscribe to `{Metadata.Name}.port.input1`, `{Metadata.Name}.port.input2`, `{Metadata.Name}.port.input3`. On each subscription callback, store `_receivedInputs[portName] = evt.Payload` then call ExecuteAsync.
- ExecuteAsync:
  1. If `_receivedInputs` is empty, return (nothing to join)
  2. Read config for `separator` key (default empty string)
  3. Join all received values (ordered by key: input1, input2, input3) using separator: `string.Join(separator, _receivedInputs.OrderBy(kv => kv.Key).Select(kv => kv.Value))`
  4. Publish joined result to `{Metadata.Name}.port.output`
  5. Clear `_receivedInputs` after publishing (ready for next execution cycle)

**3. TextSplitModule.cs**
- Ports: `[InputPort("input", PortType.Text)]`, `[OutputPort("output", PortType.Text)]`
- Metadata: `new ModuleMetadataRecord("TextSplitModule", "1.0.0", "Splits text by delimiter into JSON array")`
- InitializeAsync: Subscribe to `{Metadata.Name}.port.input`. On callback, store payload in `_pendingInput` and call ExecuteAsync.
- ExecuteAsync:
  1. If `_pendingInput` is null, return
  2. Read config for `delimiter` key (default `,`)
  3. Split input: `_pendingInput.Split(delimiter)` — use `string[]` overload: `_pendingInput.Split(new[] { delimiter }, StringSplitOptions.None)`
  4. Serialize parts to JSON array: `System.Text.Json.JsonSerializer.Serialize(parts)`
  5. Publish JSON array string to `{Metadata.Name}.port.output`

**4. ConditionalBranchModule.cs**
- Ports: `[InputPort("input", PortType.Text)]`, `[OutputPort("true", PortType.Text)]`, `[OutputPort("false", PortType.Text)]`
- Metadata: `new ModuleMetadataRecord("ConditionalBranchModule", "1.0.0", "Routes input to true/false branch based on expression")`
- InitializeAsync: Subscribe to `{Metadata.Name}.port.input`. On callback, store payload in `_pendingInput` and call ExecuteAsync.
- ExecuteAsync:
  1. If `_pendingInput` is null, return
  2. Read config for `expression` key (default empty string)
  3. If expression is empty, route to `false` output (safe default)
  4. Evaluate expression against input using `EvaluateExpression(expression, _pendingInput)`
  5. If result is true: publish `_pendingInput` to `{Metadata.Name}.port.true`
  6. If result is false: publish `_pendingInput` to `{Metadata.Name}.port.false`
  7. Wrap evaluation in try/catch — on parse error, log warning, set state to Error, route to `false` branch
- Implement `private bool EvaluateExpression(string expression, string inputValue)` as a simple recursive evaluator:
  - Handle `||` by splitting on `||` (respecting nesting with parentheses or using simple left-to-right evaluation)
  - Handle `&&` by splitting on `&&`
  - Handle `!` prefix (negation)
  - Handle parenthesized sub-expressions `(...)` by finding matching parens and recursing
  - Handle method calls: `input.contains("...")`, `input.startsWith("...")`, `input.endsWith("...")` — extract the string argument between quotes
  - Handle `input.length` comparisons: `input.length > N`, `input.length < N`, `input.length >= N`, `input.length <= N`, `input.length == N`, `input.length != N`
  - Handle equality: `input == "..."`, `input != "..."`
  - Throw `ArgumentException` for unrecognized expressions (caught by caller)
  - Keep implementation pragmatic (~80-120 lines) — do NOT over-engineer. The operator set is small and well-defined per CONTEXT.md.

All four modules follow standard patterns:
- `_state` field initialized to `ModuleExecutionState.Idle`
- `_lastError` field initialized to `null`
- Set `_state = Running` at start of execute, `Completed` on success, `Error` on failure
- ShutdownAsync disposes all subscriptions
- GetState() and GetLastError() return respective fields

With:
```csharp
/// <summary>
/// All module types whose ports should be registered in PortRegistry.
/// Includes HeartbeatModule so it appears in ModulePalette for manual addition.
/// </summary>
private static readonly Type[] PortRegistrationTypes =
{
    typeof(LLMModule),
    typeof(ChatInputModule),
    typeof(ChatOutputModule),
    typeof(HeartbeatModule),
    typeof(FixedTextModule),
    typeof(TextJoinModule),
    typeof(TextSplitModule),
    typeof(ConditionalBranchModule)
};

/// <summary>
/// Module types that are auto-initialized at startup.
/// HeartbeatModule excluded — BUILTIN-10 requires it to be optional.
/// </summary>
private static readonly Type[] AutoInitModuleTypes =
{
    typeof(LLMModule),
    typeof(ChatInputModule),
    typeof(ChatOutputModule),
    typeof(FixedTextModule),
    typeof(TextJoinModule),
    typeof(TextSplitModule),
    typeof(ConditionalBranchModule)
};
```

Then update the two methods that iterate over `ModuleTypes`:
- `RegisterModulePorts()`: change `foreach (var moduleType in ModuleTypes)` to `foreach (var moduleType in PortRegistrationTypes)`
- `InitializeModulesAsync()`: change `foreach (var moduleType in ModuleTypes)` to `foreach (var moduleType in AutoInitModuleTypes)`

This ensures HeartbeatModule's ports are registered (so it appears in ModulePalette and can be wired) but it is NOT auto-initialized at startup (satisfying BUILTIN-10: heartbeat is optional).
