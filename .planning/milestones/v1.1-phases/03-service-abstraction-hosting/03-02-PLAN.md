# Plan 03-02: Blazor Layout Shell & Browser Auto-Launch

**Phase:** 3 — Service Abstraction & Hosting
**Requirements:** INFRA-01 (complete), INFRA-03
**Depends on:** Plan 03-01

## Goal

Create the Blazor Server layout shell with dark theme, collapsible sidebar, and logo placeholder. Implement browser auto-launch on startup with --no-browser CLI flag support. Create a placeholder dashboard page.

## Tasks

### 1. Create Blazor infrastructure files
- `Components/_Imports.razor` — global using directives
- `Components/App.razor` — root component with HTML head, body, Blazor script
- `Components/Routes.razor` — Router component

### 2. Create layout with dark theme
- `Components/Layout/MainLayout.razor` — sidebar + content area
- `Components/Layout/MainLayout.razor.css` — scoped dark theme styles
- `wwwroot/css/app.css` — global CSS variables and reset

### 3. Create placeholder dashboard page
- `Components/Pages/Dashboard.razor` — "/" route with placeholder content
- Shows "OpenAnima Runtime Dashboard" heading
- Placeholder cards for modules and heartbeat (filled in Phase 4)

### 4. Implement browser auto-launch
- In Program.cs: register IHostApplicationLifetime callback
- On ApplicationStarted: open system default browser to dashboard URL
- Check args for "--no-browser" flag to skip auto-open
- Console output: show URL and "Press Ctrl+C to stop"

### 5. Add static assets
- `wwwroot/favicon.ico` — placeholder (can be updated later)
