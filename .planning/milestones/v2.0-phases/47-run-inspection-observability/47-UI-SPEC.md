## Propagation Chain Color Palette

Steps sharing the same PropagationId get a colored left border (3px) and dot indicator. Colors cycle through a fixed palette of 8 hues, assigned by index (stable across re-renders via deterministic hash of PropagationId string).

| Slot | Hex | Description |
|------|-----|-------------|
| 0 | #6c8cff | Accent blue (matches `--accent-color`) |
| 1 | #4ade80 | Success green |
| 2 | #fbbf24 | Warning amber |
| 3 | #f87171 | Error red |
| 4 | #a78bfa | Purple |
| 5 | #34d399 | Teal |
| 6 | #fb923c | Orange |
| 7 | #60a5fa | Sky blue |

Cycling strategy: `colorIndex = Math.Abs(propagationId.GetHashCode()) % 8`. Stable within a page load; consistent across SignalR appends because the same PropagationId always maps to the same slot.

Highlighted chain: full opacity left border + `--active-bg` row background (`rgba(108,140,255,0.12)`).
Dimmed steps (not in active chain): `opacity: 0.35` on the entire row.
No active chain selected: all rows at full opacity.

## Interaction Contracts

### Navigation
- RunCard click → `Navigation.NavigateTo($"/runs/{run.RunId}")` — entire card is clickable
- RunDetail back link → `Navigation.NavigateTo("/runs")` — top-left, "← Runs" label
- Browser back button works naturally (standard Blazor navigation)

### Accordion
- Click collapsed StepTimelineRow → expand (one row open at a time OR multiple — discretion: allow multiple open simultaneously, no forced close)
- Click expanded StepTimelineRow header → collapse
- Expand/collapse is local component state, no URL change

### Propagation Chain Highlight
- Click any step row → set `_activeChainId = step.PropagationId`
- All rows with matching PropagationId: full opacity + `--active-bg` background
- All other rows: `opacity: 0.35`
- Click same row again OR click empty space in timeline → clear `_activeChainId`, restore all rows to full opacity
- Highlight state is page-local, not persisted

### Propagation Chain Filter Shortcut
- In expanded step detail, "Propagation chain: [id] — N steps" line
- Clicking the PropagationId link sets the Chain filter dropdown to that PropagationId
- This is a filter activation, not a highlight — it hides non-matching rows

### Real-time Updates
- SignalR `ReceiveStepCompleted`: append new StepTimelineRow to bottom of timeline, call `InvokeAsync(StateHasChanged)`
- SignalR `ReceiveRunStateChanged`: update overview RunStateBadge + append StateEventRow to timeline
- New rows animate in: `opacity 0 → 1` over 200ms (CSS transition)
- No auto-scroll to bottom — user controls scroll position

### Long Timeline Handling
- Render all rows (no virtual scrolling for v47 — typical runs are bounded by MaxSteps)
- Timeline container: `overflow-y: auto`, `max-height: calc(100vh - 320px)`
- If step count exceeds 200: show info banner "Showing all N steps. Use filters to narrow the view." (12px, `--text-secondary`)

### Disabled Artifact Link
- "View full content ↗" renders as `<a>` with `aria-disabled="true"`, `tabindex="-1"`, `pointer-events: none`
- Color: `--accent-color` at `opacity: 0.4`
- Title attribute: "Artifact viewer coming in a future release"

## Accessibility

- All interactive rows: `role="button"`, `tabindex="0"`, `@onkeydown` handles Enter/Space for expand and highlight
- Filter selects: `<label>` elements associated via `for`/`id`
- RunStateBadge: existing `aria-label="Run state: {state}"` — no change
- Disabled artifact link: `aria-disabled="true"`, `tabindex="-1"`
- Propagation chain filter shortcut link: `aria-label="Filter by propagation chain {id}"`
- Timeline list: `role="list"` on container, `role="listitem"` on each row
- State event rows: `aria-hidden="false"`, descriptive text sufficient for screen readers
- Color is never the sole differentiator — status badges carry text labels alongside color
