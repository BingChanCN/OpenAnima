## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline gaps between status dot and label |
| sm | 8px | Button internal padding (vertical), compact row spacing within run list |
| md | 16px | Button internal padding (horizontal), card internal padding, form field gaps |
| lg | 24px | Section padding, page header margin-bottom |
| xl | 32px | Major section gaps within a run detail view |
| 2xl | 48px | Page-level top padding |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Run status badge horizontal padding: 8px (xs token — tight badge pill using standard grid value)
- Stop-reason banner padding: inherits from `.error-inline` CSS class in `app.css` — no new spacing declaration; executor must not override this class
- Budget progress indicator height: 4px track (xs token)

## Typography

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 14px | 400 | 1.5 | sans | Run list rows, form labels, step count, stop reason text |
| Label | 12px | 400 | 1.5 | sans | Secondary metadata (run ID, timestamps, module name in stop reason) |
| Heading | 20px | 600 | 1.2 | sans | Page title "Runs", run detail heading |
| Mono | 14px | 400 | 1.5 | mono | Run ID (8-char hex), step counts, duration values (e.g. "1 234 ms") |

Source: `html, body` at 14px/1.5 in `app.css`. Heading 20px matches `.page-title` pattern from Monitor/Dashboard pages. Mono class for IDs and metrics matches `.mono` utility. Label at 12px provides a 2px step below body (14px) for meaningful visual hierarchy; color `--text-secondary` additionally differentiates secondary metadata from primary content.

## Color

All values from `app.css` `:root` tokens. No new colors introduced.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--bg-primary` | `#0f1117` | Page background, run list background |
| Secondary (30%) | `--surface-card` / `--surface-dark` | `#1c1e2e` / `#161822` | Run cards, launch dialog, run detail panel |
| Accent (10%) | `--accent-color` | `#6c8cff` | Primary CTA only: "Start Run" button, "Resume Run" button, active run ID link |
| Success | `--success-color` | `#4ade80` | Running state badge, completed state badge |
| Warning | `--warning-color` | `#fbbf24` | Paused state badge, Interrupted state badge, budget-warning indicator |
| Destructive | `--error-color` | `#f87171` | Cancel run button, Failed state badge, stop-reason banner border |
| Text primary | `--text-primary` | `#e4e6f0` | All primary content text |
| Text secondary | `--text-secondary` | `#8b8fa3` | Run ID, timestamps, secondary metadata (Label role) |
| Border | `--border-color` | `#2a2d3e` | Card borders, form field borders, dividers |

**Accent reserved for:** "Start Run" button (primary CTA), "Resume Run" button, run ID in active state (clickable link). Never used for status badges, metadata, or secondary actions.

**Primary focal point:** The RunLaunchPanel "Start Run" button — the only element on the page using the accent color (`--accent-color` / `#6c8cff`). All other interactive elements use neutral or semantic colors. This anchors the user's attention to the primary action before browsing run history.

| Run State | Badge Color Token | Rationale |
|-----------|------------------|-----------|
| Running | `--success-color` (#4ade80) | Active, healthy |
| Completed | `--success-color` (#4ade80) | Finished successfully |
| Paused | `--warning-color` (#fbbf24) | Attention needed — can resume |
| Interrupted | `--warning-color` (#fbbf24) | Recoverable — needs user action |
| Cancelled | `--text-muted` (#5c5f73) | Terminal, neutral |

## Interaction States

### Run Lifecycle — User-triggered transitions

| Action | Enabled When | Result |
|--------|-------------|--------|
| Start Run | Form valid (objective non-empty, workspace non-empty) | Creates run, shows in list with Running badge via SignalR push |
| Pause Run | Run is in Running state | Transitions to Paused, updates badge in real-time |
| Resume Run | Run is in Paused or Interrupted state | Transitions to Running, updates badge in real-time |
| Cancel Run | Run is in Running, Paused, or Interrupted state | Shows `ConfirmDialog`, on confirm transitions to Cancelled |

### System-triggered state changes (display only, no user action)

| Event | Visual Change |
|-------|--------------|
| Budget exhausted (CTRL-01) | Badge → Paused (warning), `StopReasonBanner` appears below run card with reason text |
| Non-productive pattern (CTRL-02) | Badge → Paused (warning), `StopReasonBanner` appears with reason text |
| Run completes normally | Badge → Completed (success) |
| Run fails with error | Badge → Failed (error) |
| Recovery on startup (RUN-02) | Badge → Interrupted (warning) for affected runs |

### Form validation

| Field | Validation | Error display |
|-------|-----------|--------------|
| Objective | Required, non-empty | Inline `.error-inline` below field |
| Workspace Root | Required, non-empty | Inline `.error-inline` below field |
| Max Steps | Optional; if provided, must be integer > 0 | Inline `.error-inline` below field |
| Max Wall Time (s) | Optional; if provided, must be integer > 0 | Inline `.error-inline` below field |

Start Run button: disabled while form is invalid or while a submit is in-flight (spinner inside button per `.btn.loading` pattern).
