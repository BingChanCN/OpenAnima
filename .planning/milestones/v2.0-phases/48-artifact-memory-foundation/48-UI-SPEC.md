## Layout Specification

### ArtifactViewer (inline in StepTimelineRow accordion)

Rendered below the existing input/output/error sections in the expanded step accordion. Only rendered when `ArtifactRefId` is non-null.

```
┌─────────────────────────────────────────────────────┐
│  ARTIFACT                                            │  section label, 12px, text-secondary
│  ┌───────────────────────────────────────────────┐  │
│  │  [MIME badge]  filename.ext  [size]  [ts]     │  │  surface-dark, 8px padding
│  │  ─────────────────────────────────────────── │  │
│  │  [rendered content — see MIME rules below]   │  │  surface-card, 12px padding
│  │                                               │  │
│  │  [查看完整内容 ↓]  (only if truncated)        │  │  accent link, 12px
│  └───────────────────────────────────────────────┘  │
│  Provenance: Step [stepId link] · Run [runId link]  │  12px, text-secondary, accent links
└─────────────────────────────────────────────────────┘
```

MIME rendering rules:
- `text/markdown` → rendered as HTML via Markdown-to-HTML (server-side, no JS dependency). Headings, bold, code blocks preserved.
- `application/json` → collapsible JSON tree. Top-level keys shown expanded; nested objects collapsed by default. Toggle per node.
- `text/plain` → `<pre>` block, mono font, 12px, `--surface-dark` background, 8px padding, `overflow-x: auto`.
- `text/html` → plain text fallback (no iframe, no raw HTML render for security). Display as `text/plain` with MIME label showing "text/html (plain)".
- Unknown MIME → plain text fallback with MIME label.

Truncation: first 200 lines or 10KB, whichever is smaller. Truncation notice: "Showing first {N} lines of {total} — 查看完整内容" in `--text-muted` 12px. Expand loads full content in-place (no navigation).

### MemoryGraph Page Layout

Focal point: the selected `MemoryNodeCard` in the right detail panel is the primary focal point. It occupies 70% of the horizontal space, sits at eye level on load, and is the only editable surface on the page. The URI tree is a navigation aid — visually subordinate (lower contrast, smaller type, narrower column).

```
┌─────────────────────────────────────────────────────┐
│  Memory                              [+ New Node]   │  page title + primary CTA
├─────────────────────────────────────────────────────┤
│  [URI search input]                                  │  filter bar, full width
├──────────────────────┬──────────────────────────────┤
│  URI TREE            │  NODE DETAIL  ← focal point  │
│  (30% width)         │  (70% width)                 │
│                      │                              │
│  core://             │  [MemoryNodeCard — selected] │
│    agent/            │                              │
│      identity  ●     │  URI: core://agent/identity  │
│      mission         │  Content: [editable textarea]│
│  run://              │  Trigger: [editable input]   │
│    abc123/           │  Keywords: [tag input]       │
│      findings        │  Provenance: step · artifact │
│  project://          │                              │
│    myapp/            │  [Save]  [Delete]            │
│      architecture    │                              │
└──────────────────────┴──────────────────────────────┘
```

- URI tree: left panel, `--surface-dark` background, 16px padding. Tree nodes are indented 16px per depth level. Leaf nodes show a filled dot (●, 8px, `--accent-color`) when selected, empty circle (○) otherwise.
- Node detail: right panel, `--surface-card` background, 16px padding. Shows full `MemoryNodeCard` for selected node in edit mode.
- No node selected: right panel shows empty state — "Select a memory node to view or edit."
- Responsive: on narrow viewports (< 768px), tree and detail stack vertically; tree collapses to a flat list.

## Interaction Contracts

### Artifact Viewer

- Artifact section in step accordion: rendered only when `ArtifactRefId != null`. Hidden (no placeholder) when null.
- "查看完整内容" expand: loads full artifact content via service call, replaces truncated preview in-place. Button changes to "收起" (Collapse) after expand. No navigation.
- JSON tree toggle: click any object/array node label to expand/collapse. State is local to the component instance.
- Provenance links: `--accent-color` text, `cursor: pointer`. Click stepId link → `Navigation.NavigateTo($"/runs/{runId}#step-{stepId}")`. Click runId link → `Navigation.NavigateTo($"/runs/{runId}")`.

### Memory Graph Page

- URI tree node click → select node, load detail in right panel. Selected node highlighted with `--active-bg` background row.
- "+ New Node" button → opens inline form in right panel with empty URI input, content textarea, trigger input. URI field focused on open.
- Save Changes → `POST` to memory service, optimistic UI update of tree label. Show inline success flash (green border, 1.5s, then fade) on the node card.
- Delete Node → opens `ConfirmDialog`. Confirmation copy: "Delete memory node?" / "This will permanently remove the node and all its edges. This cannot be undone." / Confirm button: "Delete Node" (`btn-danger`).
- URI search input → filters tree to matching nodes in real-time (client-side string match on URI path). Clears with × button.
- Keyword tag add: type in "+ add keyword" input, press Enter or comma to add tag. Click × on existing tag to remove.

### Memory Injection Step Records

- Memory injection steps appear in the run timeline (Phase 47 `StepTimelineRow`) with module name "MemoryModule".
- Expanded detail shows: injected memory URI(s) as accent-colored pills, source artifact provenance link, disclosure trigger condition text in `--text-muted`.
- No new component needed — handled by existing `StepTimelineRow` accordion with artifact viewer extension.

### System Boot Steps

- Boot memory injection steps appear at the top of the run timeline with module name "BootMemory" and a distinct left border color: slot 4 (`#a78bfa`, purple) from the Phase 47 propagation color palette.
- No new component needed.

## Accessibility

- `ArtifactViewer`: artifact section has `role="region"` with `aria-label="Artifact: {filename}"`. JSON tree toggle buttons have `aria-expanded` state. Expand/collapse button has descriptive `aria-label`.
- `MemoryNodeCard`: form fields have associated `<label>` elements. Delete button has `aria-label="Delete memory node {uri}"`. Save button has `aria-label="Save changes to {uri}"`.
- URI tree: `role="tree"` on container, `role="treeitem"` on each node, `aria-selected` on selected node, `aria-expanded` on expandable path segments.
- Keyword tags: each tag has a remove button with `aria-label="Remove keyword {tag}"`.
- Provenance links: `aria-label="View step {stepId}"` and `aria-label="View run {runId}"`.
- Color is never the sole differentiator — MIME type badges carry text labels, status indicators carry text alongside color.
- `ConfirmDialog` (delete): focus trapped within dialog while open. Escape key dismisses.
