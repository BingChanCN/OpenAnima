### Schema Mismatch (Tech Debt)

**Problem:** CLI generates extended manifest schema, but runtime only parses subset.

**CLI ModuleManifest** (`src/OpenAnima.Cli/Models/ModuleManifest.cs`):
- ✓ schemaVersion
- ✓ id
- ✓ name
- ✓ version
- ✓ description
- ✓ author
- ✓ entryAssembly
- ✓ openanima (minVersion, maxVersion)
- ✓ ports (inputs, outputs)
- ✓ targetFramework
- ✓ checksum (algorithm, value)

**Runtime PluginManifest** (`src/OpenAnima.Core/Plugins/PluginManifest.cs`):
- ✓ name
- ✓ version
- ✓ description
- ✓ entryAssembly
- ✗ **Missing:** id, schemaVersion, author, openanima, ports, targetFramework, checksum

**Impact:**
- Port declarations generated but not used (MAN-03 tech debt)
- Version compatibility checks not enforced (MAN-02 tech debt)
- Checksum verification not performed (PACK-05 tech debt)
- Extended metadata (id, author) silently ignored (MAN-01 tech debt)

**Affected Requirements:** MAN-01, MAN-02, MAN-03, MAN-05, PACK-05, PACK-06

**Functional Impact:** Basic create→pack→load flow works. Modules load and run correctly. Advanced features (port declarations, version compatibility, checksum verification) are generated but not consumed.

**Severity:** Medium - Non-blocking for v1.4 completion. Advanced features can be added in future milestone.
