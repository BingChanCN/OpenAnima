**Step 1: Add GetPortTooltip helper method** in the @code block, after the existing GetDisplayName method (after line 205). The method signature and logic:

```csharp
private string GetPortTooltip(PortMetadata port)
{
    // HttpRequestModule has duplicate "body" port name for input and output
    // Use direction qualifier for disambiguation
    var portKey = (port.ModuleName == "HttpRequestModule" && port.Name == "body")
        ? $"Port.Description.{port.ModuleName}.{port.Name}.{port.Direction}"
        : $"Port.Description.{port.ModuleName}.{port.Name}";
    var localized = L[portKey];
    if (localized.ResourceNotFound)
        return $"{port.Name} ({port.Type})";
    return $"{port.Name}: {localized.Value} ({port.Type})";
}
```

Key behaviors:
- For HttpRequestModule "body" port: looks up `Port.Description.HttpRequestModule.body.Input` or `Port.Description.HttpRequestModule.body.Output` based on port.Direction
- For all other ports: looks up `Port.Description.{ModuleName}.{PortName}`
- If .resx key NOT found (ResourceNotFound): returns `"{port.Name} ({port.Type})"` (fallback format, no description)
- If .resx key found: returns `"{port.Name}: {localized.Value} ({port.Type})"` (full format with description)
- port.Type renders as PortType enum name (Text or Trigger) — not localized
