**src/OpenAnima.Core/Channels/ActivityChannelHost.cs:**
- Namespace: `OpenAnima.Core.Channels`
- `internal sealed class ActivityChannelHost : IAsyncDisposable`
- Constructor takes:
  - `ILogger<ActivityChannelHost> logger`
  - `Func<TickWorkItem, Task> onTick` — callback invoked by heartbeat consumer (Plan 02 will pass WiringEngine.ExecuteAsync wrapper)
  - `Func<ChatWorkItem, Task> onChat` — callback for chat consumer (Plan 02 will pass EventBus.PublishAsync wrapper)
  - `Func<RouteWorkItem, Task> onRoute` — callback for routing consumer (Plan 02 will pass EventBus.PublishAsync wrapper)
- Three private `Channel<T>` fields, each created with `Channel.CreateUnbounded<T>(new UnboundedChannelOptions { SingleReader = true })`
- Private `CancellationTokenSource _cts` for shutdown safety
- Private `Task? _heartbeatConsumer, _chatConsumer, _routingConsumer`
- Private `long _coalescedTickCount` for tracking coalesced ticks (Interlocked access)
- Public `long CoalescedTickCount => Interlocked.Read(ref _coalescedTickCount)`

**Stateless dispatch helper (CONC-08 building block):**
- Private static field: `private static readonly ConcurrentDictionary<Type, bool> _statelessCache = new();`
- Public static method: `public static bool IsStateless(IModule module)` — checks `module.GetType().GetCustomAttribute<StatelessModuleAttribute>() != null` via `_statelessCache.GetOrAdd(type, t => ...)`. Returns cached bool.
- This method is the single entry point for stateless classification. Plan 02 uses it in the onTick callback to fork dispatch between concurrent (stateless) and serialized (stateful) module execution.
- Static so that the cache is shared across all ActivityChannelHost instances (module types don't change at runtime).

**Public methods:**
- `void Start()` — starts 3 background consumer Tasks via `Task.Run`. Each calls `ConsumeXxxAsync(_cts.Token)`.
- `void EnqueueTick(TickWorkItem item)` — `_heartbeatChannel.Writer.TryWrite(item)` (void return, always succeeds on unbounded)
- `void EnqueueChat(ChatWorkItem item)` — `_chatChannel.Writer.TryWrite(item)` + queue depth warning if `Reader.Count > 10`
- `void EnqueueRoute(RouteWorkItem item)` — `_routingChannel.Writer.TryWrite(item)` + queue depth warning if `Reader.Count > 10`

**Private consumer methods:**
- `ConsumeHeartbeatAsync(CancellationToken ct)`:
  - Use `await _heartbeatChannel.Reader.ReadAsync(ct)` to get first item
  - Then coalesce: `while (_heartbeatChannel.Reader.TryRead(out var next)) { item = next; coalescedCount++; }`
  - If coalescedCount > 0: `Interlocked.Add(ref _coalescedTickCount, coalescedCount)` + LogWarning
  - Call `await _onTick(item)` with try/catch (log error, don't crash consumer)
  - Loop: wrap in `while (!ct.IsCancellationRequested)` with outer try for OperationCanceledException + ChannelClosedException
- `ConsumeChatAsync(CancellationToken ct)`:
  - `await foreach (var item in _chatChannel.Reader.ReadAllAsync(ct))` — call `await _onChat(item)` with try/catch
- `ConsumeRoutingAsync(CancellationToken ct)`:
  - `await foreach (var item in _routingChannel.Reader.ReadAllAsync(ct))` — call `await _onRoute(item)` with try/catch

**Const:**
- `private const int QueueDepthWarningThreshold = 10;`

**DisposeAsync:**
- Complete all 3 writers: `_heartbeatChannel.Writer.Complete()`, etc.
- Cancel CTS as safety net
- `await Task.WhenAll(filter nulls from [_heartbeatConsumer, _chatConsumer, _routingConsumer])`
- Dispose CTS
