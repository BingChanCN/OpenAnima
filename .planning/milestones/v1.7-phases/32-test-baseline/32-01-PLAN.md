From src/OpenAnima.Core/Wiring/WiringEngine.cs — CreateRoutingSubscription (DO NOT MODIFY):
```csharp
private IDisposable CreateRoutingSubscription(
    string sourceEventName, string targetEventName,
    string sourceModuleRuntimeName, PortType? sourcePortType)
{
    return sourcePortType switch
    {
        PortType.Text => _eventBus.Subscribe<string>(sourceEventName, ...),
        PortType.Trigger => _eventBus.Subscribe<DateTime>(sourceEventName, ...),
        _ => _eventBus.Subscribe<object>(sourceEventName, ...)
    };
}
```
Key insight: For PortType.Text ports, WiringEngine subscribes with `Subscribe<string>`. The EventBus uses `typeof(TPayload)` as the dispatch key. Publishing `ModuleEvent<object>` goes to the `object` bucket — the `string` subscription never fires. This is NOT a production bug; it is a test-authoring error.
