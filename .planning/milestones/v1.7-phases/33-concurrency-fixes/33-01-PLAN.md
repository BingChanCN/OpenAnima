**Pattern A: Local capture (eliminate _pending* field race) — LLMModule, ConditionalBranchModule, TextSplitModule**

For each module:
1. Delete the `_pendingPrompt` / `_pendingInput` field declaration.
2. In InitializeAsync lambda, change:
   ```csharp
   // BEFORE:
   _pendingPrompt = evt.Payload;
   await ExecuteAsync(ct);
   // AFTER:
   var prompt = evt.Payload;       // local capture
   await ExecuteInternalAsync(prompt, ct);
   ```
3. Rename the existing `ExecuteAsync` body to `private async Task ExecuteInternalAsync(string prompt, CancellationToken ct)` (or `string input` for ConditionalBranch/TextSplit).
4. Replace all reads of `_pendingPrompt` / `_pendingInput` inside the method body with the local parameter.
5. Keep the no-arg `public Task ExecuteAsync(CancellationToken ct = default) => Task.CompletedTask;` as IModuleExecutor interface compliance.

**Pattern B: SemaphoreSlim(1,1) skip-when-busy guard — All 5 modules**

For LLMModule, ConditionalBranchModule, TextSplitModule, TextJoinModule:
1. Add field: `private readonly SemaphoreSlim _executionGuard = new SemaphoreSlim(1, 1);`
2. At the top of `ExecuteInternalAsync`, add:
   ```csharp
   if (!_executionGuard.Wait(0)) return;  // skip if already running
   try
   {
       // ... existing body ...
   }
   finally
   {
       _executionGuard.Release();
   }
   ```
3. The `_state = Running` and `_state = Completed/Error` assignments stay inside the try block.
4. The catch block stays inside try (before finally).

For HttpRequestModule (different pattern — event-driven via trigger):
1. Add field: `private readonly SemaphoreSlim _executionGuard = new SemaphoreSlim(1, 1);`
2. Wrap the trigger subscription lambda:
   ```csharp
   var triggerSub = _eventBus.Subscribe<string>(
       $"{Metadata.Name}.port.trigger",
       async (evt, ct) =>
       {
           if (!_executionGuard.Wait(0)) return;
           try
           {
               await HandleTriggerAsync(ct);
           }
           finally
           {
               _executionGuard.Release();
           }
       });
   ```
3. Do NOT add the guard inside HandleTriggerAsync itself — the guard wraps it externally.

**Pattern C: ConcurrentDictionary for _receivedInputs — TextJoinModule only**

1. Add `using System.Collections.Concurrent;` at top.
2. Change `private readonly Dictionary<string, string> _receivedInputs = new();` to `private readonly ConcurrentDictionary<string, string> _receivedInputs = new();`.
3. Index-based writes (`_receivedInputs[capturedPort] = evt.Payload`) and reads remain the same — ConcurrentDictionary supports indexer assignment.
4. `.Clear()` in ExecuteInternalAsync remains the same.
5. Change `_receivedInputs.Count == 0` guard to `_receivedInputs.IsEmpty` (ConcurrentDictionary has `IsEmpty` property which is more idiomatic; `.Count == 0` also works but IsEmpty is preferred).

**TextJoinModule additional local-capture fix:**
TextJoinModule does NOT have a `_pendingInput` field — its subscription lambdas write directly to `_receivedInputs[capturedPort]` then call `ExecuteAsync`. Apply the same ExecuteInternalAsync pattern:
- The subscription lambda becomes:
  ```csharp
  async (evt, ct) =>
  {
      _receivedInputs[capturedPort] = evt.Payload;
      await ExecuteInternalAsync(ct);
  }
  ```
- Note: TextJoinModule's ExecuteInternalAsync takes no input parameter (it reads from _receivedInputs dictionary). The semaphore guard wraps ExecuteInternalAsync.
- Keep no-arg `ExecuteAsync` as interface compliance no-op.

**After all changes, run the full test suite** to verify CONC-01 (zero regressions) and CONC-04 (ConcurrencyGuardTests now pass with guards in place).

CRITICAL NOTES:
- `SemaphoreSlim.Wait(0)` is the SYNCHRONOUS non-blocking call. Do NOT use `WaitAsync()` — that queues instead of skipping.
- `Release()` MUST be in `finally` — if execution throws and Release is missed, the semaphore is permanently held.
- The no-arg `ExecuteAsync` MUST remain on all modules for IModuleExecutor interface compliance. It becomes a no-op (`=> Task.CompletedTask`).
- Do NOT add semaphore guards to modules not listed (ChatInput, ChatOutput, Heartbeat, FixedText, AnimaInputPort, AnimaOutputPort, AnimaRoute) — they have no shared mutable state in their execution paths.
