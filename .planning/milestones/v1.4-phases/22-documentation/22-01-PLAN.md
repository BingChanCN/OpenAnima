# Plan 22-01: Quick-Start Guide

**Goal:** Create a 5-minute quick-start guide showing the create-build-pack workflow with concrete examples.

**Context:** Phase 22 creates developer documentation for the OpenAnima module SDK. This plan focuses on the quick-start guide — the primary entry point for new developers. The guide must enable a developer to go from zero to working module in under 5 minutes (DOC-02). The CLI tool (oani) is complete with all commands (new, pack, validate). Built-in modules demonstrate real-world patterns. The challenge is organizing information for maximum developer velocity.

**Approach:** Write two Markdown files: (1) `docs/README.md` as the documentation index with clear "Start here" link, and (2) `docs/quick-start.md` as a single-page tutorial with step-by-step commands and expected output. Structure the quick-start as: Prerequisites → Create module (30s) → Implement logic (2m) → Build (30s) → Pack (30s) → Load in runtime (30s) = 4.5 minutes total. Show expected output after each command to confirm success. Use inline code comments instead of separate explanations. End with clear next steps linking to API reference.

**Key decisions:**
- Plain Markdown files (no DocFX/MkDocs build step for v1.4)
- "Hello World" minimal example (fast to complete, then link to real-world patterns)
- Show expected output after every command (confirms success)
- Lead with time estimate (builds confidence)
- Inline code comments (faster to read than separate explanations)
