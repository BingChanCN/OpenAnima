# Plan 22-02: API Reference Documentation

**Goal:** Create API reference documentation for all public interfaces, the port system, and common module patterns.

**Context:** Phase 22 creates developer documentation for the OpenAnima module SDK. This plan focuses on the API reference — detailed documentation for developers implementing modules. The existing codebase has excellent XML documentation comments on all interfaces (IModule, IModuleExecutor, ITickable, IEventBus, port attributes). Built-in modules (ChatInputModule, LLMModule, ChatOutputModule) demonstrate real-world port usage patterns. The challenge is organizing this information for reference lookup, not tutorial learning.

**Approach:** Create seven Markdown files in `docs/api-reference/`: (1) README.md as index, (2-5) one page per interface (IModule, IModuleExecutor, ITickable, IEventBus) with purpose/methods/lifecycle/examples, (6) port-system.md documenting PortType, attributes, and EventBus usage, (7) common-patterns.md with code examples extracted from built-in modules. Structure each interface page as: Purpose → Definition → Members → Lifecycle → See Also. Structure common-patterns.md by module topology: Source (no inputs), Transform (input→output), Sink (input only), Heartbeat (ITickable).

**Key decisions:**
- One page per interface (focused reference, not comprehensive)
- Document the 80% use case first (edge cases link to XML comments)
- Extract examples from real built-in modules (proven patterns)
- Organize common-patterns.md by topology (source/transform/sink/heartbeat)
- Plain Markdown (no DocFX generation for v1.4)
