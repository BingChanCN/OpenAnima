CrossAnimaRouter has two constructors: one taking `Lazy<IAnimaRuntimeManager>?` and one taking `IAnimaRuntimeManager?`. When tests call `new CrossAnimaRouter(logger)` with only 1 argument, compiler cannot pick which optional second param to use.

Fix: At every `new CrossAnimaRouter(NullLogger<CrossAnimaRouter>.Instance)` call site, add an explicit cast to disambiguate. Use the `Lazy<IAnimaRuntimeManager>?` overload since that is the primary constructor:
```csharp
new CrossAnimaRouter(NullLogger<CrossAnimaRouter>.Instance, (Lazy<IAnimaRuntimeManager>?)null)
```

Affected files and approximate line numbers:
- `RoutingModulesTests.cs`: lines 165, 189, 219, 249, 279, 340, 395 (7 call sites)
- `CrossAnimaRouterIntegrationTests.cs`: line 29 (1 call site)
- `CrossAnimaRouterTests.cs`: lines 17, 316, 324 (3 call sites)

Note: `CrossAnimaRoutingE2ETests.cs` line 35 passes an actual `runtimeManager` argument so it is NOT ambiguous and needs no fix.

**CS1503 — EditorStateService constructor parameter order mismatch (2 call sites):**
The current EditorStateService constructor signature is: `(IPortRegistry, IConfigurationLoader, ILogger<EditorStateService>, IWiringEngine?)`. Tests pass arguments in old order: `(portRegistry, configLoader, wiringEngine, logger)`.

Fix: Swap the 3rd and 4th arguments so logger comes before wiringEngine:
- `EditorStateServiceTests.cs` line 22-27: Change from `(portRegistry, configLoader, wiringEngine, logger)` to `(portRegistry, configLoader, logger, wiringEngine)`
- `EditorRuntimeStatusIntegrationTests.cs` line 65-69: Same swap
