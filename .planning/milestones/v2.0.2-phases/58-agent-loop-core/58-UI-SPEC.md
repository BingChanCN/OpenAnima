# Phase 58 — UI Design Contract

> Visual and interaction contract for Phase 58: Agent Loop Core.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope Note

Phase 58 is a backend infrastructure phase. No new visual surfaces are introduced.
The UI-facing changes are:

1. Two new LLMModule config keys rendered by the existing config schema system inside EditorConfigSidebar — no new component, no new layout.
2. One copy string appended to the assistant message bubble when the iteration ceiling is hit.

All tool call display work (collapsible cards, count badge, send lock) belongs to Phase 59.
This UI-SPEC documents only the contract for Phase 58's narrow UI-adjacent outputs.

## Component Inventory

No new components are introduced. The following existing components are extended:

### LLMModule config schema (EditorConfigSidebar)

Two new fields rendered by the existing config schema system:

**agentEnabled** (bool)
- Rendered as: existing `<input type="checkbox">` pattern within `.config-field`
- Label: "Agent Mode" — 0.875rem, weight 400, color `--text-secondary`
- Hint: "When enabled, the LLM will parse tool calls and loop until the task is complete." — 0.75rem, color `--text-secondary`, margin-top 4px
- No destructive action. No confirmation required.

**agentMaxIterations** (int)
- Rendered as: existing `<input type="number">` pattern within `.config-field`
- Label: "Max Iterations" — 0.875rem, weight 400, color `--text-secondary`
- Hint: "Maximum tool call iterations per response (default 10, server ceiling 50)." — 0.75rem, color `--text-secondary`
- Warning inline when value > 20: "Large iteration counts may produce long response times." — `warning-inline` class, `--warning-color` styling (pre-existing CSS rule)
- Validation error when value < 1 or > 50: "Must be between 1 and 50." — `.field-error` class, `--error-color`

## Interaction States

### agentEnabled checkbox
- Unchecked (default): agentMaxIterations field visible but visually de-emphasized — opacity 0.5, `cursor: not-allowed` on the input, label reads "Max Iterations (agent mode off)"
- Checked: agentMaxIterations field at full opacity, normal cursor

### agentMaxIterations input
- Default: shows 10
- Focused: `border-color: var(--accent-color)` — matching existing `.config-field input:focus` rule
- Value 1–20: no warning
- Value 21–50: `warning-inline` block shown below the field
- Value outside 1–50: `field-error` shown, save button blocked

### Iteration limit reached text
- Appended as plain text to the assistant bubble content, separated by a newline
- Styled identically to the rest of the message body — no special color, no badge
- This is a Phase 58 contract; Phase 59 may choose to render it differently
