## Scope Note

Phase 60 is predominantly a backend hardening phase. The only visible UI surface change is in the **Run inspector timeline** (`RunDetail.razor` / `StepTimelineRow.razor`): the timeline gains iteration bracket rows for agent loops. No new pages, dialogs, modals, or navigation elements are introduced.

UI work in this phase is limited to:
1. **AgentLoop bracket row** — a wrapper row in the timeline representing the full agent loop duration
2. **AgentIteration #N parent rows** — one collapsible row per agent loop iteration
3. **Child tool call rows** — indented child rows nested under each iteration row

### Visual treatment contract for bracket steps

The executor MUST apply these CSS rules to distinguish bracket rows from regular step rows. These extend the existing `.step-row` class — no separate component is required unless the executor determines a new component is cleaner.

**AgentLoop bracket row** (`ModuleName` = "AgentLoop"):
```css
.step-row.agent-loop-bracket {
    background: var(--surface-dark);
    border-top: 1px solid var(--border-color);
    border-bottom: 1px solid var(--border-color);
}
```

**AgentIteration parent row** (`ModuleName` starts with "AgentIteration"):
```css
.step-row.agent-iteration {
    padding-left: 24px;
    background: var(--surface-card);
    border-left-style: dashed; /* distinguishes from regular solid-border steps */
}
```

**Child tool call rows under an iteration** (identified by being child steps of an AgentIteration PropagationId):
```css
.step-row.agent-tool-call {
    padding-left: 40px;
    background: var(--bg-primary);
}
```

Nesting is expressed by left-padding only — no tree lines or connector SVGs needed for this phase.

## Interaction Contract

### Timeline iteration brackets

- The AgentLoop bracket row is **always collapsible** using the existing `step-row` toggle pattern (click or Enter/Space on keyboard)
- When collapsed: bracket row shows `Agent Loop`, iteration count (from outputSummary), total duration
- When expanded: child AgentIteration rows are visible; each child is independently collapsible
- AgentIteration rows follow the same expand/collapse pattern as `StepTimelineRow` — click to show inputSummary (first 200 chars of LLM response) and outputSummary
- Child tool call rows expand to show tool name, parameters (inputSummary), result (outputSummary), and error if present
- PropagationId chain filter works the same way: clicking a chain link highlights all steps sharing that propagation ID, including across bracket boundaries

### Token budget config field

- `agentContextWindowSize` renders as a standard integer input in the LLMModule config sidebar (`EditorConfigSidebar.razor`) under the existing `agent` config group
- No new UI affordance: the existing `GetSchema()` integer field pattern handles this automatically
- No validation UI required beyond the existing schema validation pipeline

### Sedimentation (no UI surface)

- Sedimentation changes are entirely backend. No UI confirmation, progress indicator, or error surface is added for this phase.

## Accessibility

- All new bracket rows follow existing `step-row` pattern: `role="button"`, `tabindex="0"`, keyboard handler for Enter and Space
- Iteration count badge: wrap in `<span aria-label="N iterations">` so screen readers announce the count
- No new ARIA landmark regions required
- Color is never the sole indicator of status: status icons (✓ ✗ ● ○) are used alongside color classes
